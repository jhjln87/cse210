1. The Data Layer (Inheritance & Polymorphism)


MetricSource (Abstract Base Class): Similar to your Activity or Goal class, this represents any metric your PC can measure. It will have private fields for _metricName and _currentValue (Encapsulation). It defines an abstract method: public abstract void RefreshValue();

CpuMetric (Derived Class): Inherits from MetricSource. It overrides RefreshValue() to poll Windows for live CPU utilization using C#'s System.Diagnostics.PerformanceCounter.

RamMetric (Derived Class): Inherits from MetricSource. It overrides RefreshValue() to calculate available system memory.

(Optional addition: NetworkMetric or GpuMetric to easily expand options later).


2. The Presentation Layer (Polymorphism & Abstraction)


DisplayLayout (Abstract Base Class): Defines how data should look on the small hardware screen. It contains a virtual or abstract method: public abstract string FormatData(List<MetricSource> metrics);

SummaryLayout (Derived Class): Inherits from DisplayLayout. Overrides FormatData() to create a dense text view showing multiple metrics at once (e.g., CP:45%|RM:32%).

FocusLayout (Derived Class): Inherits from DisplayLayout. Overrides FormatData() to look closely at just one specific metric, perhaps adding a calculated text-based progress bar (e.g., CPU [████░░░░░░] 40%).


3. The Infrastructure & Coordination Layer


SerialHardwareInterface (Class): Encapsulates the actual serial communications. It handles opening/closing the connection (System.IO.Ports.SerialPort) and has a clean public method like public void SendDataPacket(string payload). It hides the ugly details of baud rates and port configurations.

MonitorEngine (Class): This is the brains, mimicking your Program class's loop framework. It maintains a List<MetricSource> and the active DisplayLayout. It runs a timer or continuous loop that refreshes the metrics, feeds them to the layout, and hands the resulting string to the SerialHardwareInterface.

ProfileManager (Class): Similar to how you handled file loading/saving in your goals app, this class reads a local configuration file (e.g., config.txt) to remember user choices, such as which layout profile to load on startup or what refresh rate to execute.
